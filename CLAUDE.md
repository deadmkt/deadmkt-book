# deadmkt-book

Documentation site for the deadmkt protocol.

## Structure

- `docs/` — VitePress source (markdown). Edit these files.
- `docs/public/` — Static assets copied to build output as-is.
- `public/` — Main website (hand-coded HTML: landing page, trades dashboard).
- `public/docs/` — Built VitePress output. Do NOT edit — regenerated by build.

## Run

```bash
npm install

# Dev server (VitePress hot-reload for docs/)
npx vitepress dev docs

# Production build (outputs to public/docs/)
npx vitepress build docs

# Serve the full site (static server)
node server.js
```

## Rules

- Local git only (no push to remotes)
- Edit in `docs/`, never in `public/docs/` (build output)
- No non-ASCII characters in content files
