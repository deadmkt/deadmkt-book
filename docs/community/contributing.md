# Contributing

DeadMKT is open source. Contributions are welcome — whether that's code, strategies, documentation, or feedback.

## Code

The node software is at [github.com/deadmkt](https://github.com/deadmkt). To contribute:

1. Fork the repository
2. Create a feature/branch off the develop branch for your change
3. Submit a pull request with a clear description of what you changed and why

The codebase is Rust. If you're not a Rust developer, that's fine — there are other ways to contribute and tested AI revisions are welcome too.

## Openclaw Skill

This Infrastructure was built for AI, here is an example skill (provided without warranty and updates to come).

Strategy files don't need to be in Python — anything that speaks WebSocket and JSON works. Go, JavaScript, Rust, whatever you prefer.

## Documentation

Found a typo? Something unclear? The documentation source is in the `public/docs/` directory and uses Markdown. Edit a file, submit a PR.

## Feedback

Found a bug? Have a feature request? Open an issue on [GitHub](https://github.com/deadmkt).

A dedicated community forum is coming soon for longer discussions.

## Guidelines

- Be honest. This project values transparency above all.
- Be helpful. We're building something together.
- Don't game the system. The protocol is designed to make cheating visible and costly. The community should be too.
